Introduction

Most papers about neural networks use the notation of vectors and matrices from applied linear algebra. This notation is very well-suited to talking about vector spaces, but less well-suited to talking about neural networks. Consider the following equation (Vaswani et al. 2017): \[\text{Attention}(Q, K, V) = \softmax \left( \frac{QK^\top}{\sqrt{d_k}} \right) V.\] where \(Q\), \(K\), and \(V\) are sequences of query, key, and value vectors packed into matrices. Does the product \(QK^\top\) sum over the sequence, or over the query/key features? We would need to know the sizes of \(Q\), \(K\), and \(V\) to know that it’s taken over the query/key features. Is the softmax taken over the query sequence or the key sequence? The usual notation doesn’t even offer a way to answer this question. With multiple attention heads, the notation becomes more complicated and leaves more questions unanswered. With multiple sentences in a minibatch, the notation becomes more complicated still, and most papers wisely leave this detail out.

Libraries for programming with neural networks (Harris et al. 2020; Paszke et al. 2019) provide multidimensional arrays, called tensors (although usually without the theory associated with tensors in linear algebra and physics), and a rich array of operations on tensors. But they inherit from math the convention of identifying indices by position, making code bug-prone. Quite a few libraries have been developed to identify indices by name instead: Nexus (Chen 2017), tsalib (Sinha 2018), NamedTensor (Rush 2019), named tensors in PyTorch (Torch Contributors 2019), and Dex (Maclaurin et al. 2019). (Some of these libraries also add types to indices, but here we are only interested in adding names.)

Back in the realm of mathematical notation, then, we want two things: first, the flexibility of working with multidimensional arrays, and second, the perspicuity of identifying indices by name instead of by position. This document describes our proposal to do both.

As a preview, the above equation becomes \[\text{Attention}(Q,K,V) = \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{seq}}} \left( \frac{Q \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{key}}}} K}{\sqrt{d_k}} \right) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{seq}}}} V\] making it unambiguous which index each operation applies to. The same equation works with multiple heads and with minibatching.

More examples of the notation are given in §3.

The source code for this document can be found at https://github.com/namedtensor/notation/. We invite anyone to make comments on this proposal by submitting issues or pull requests on this repository.

Informal Overview

Let’s think first about the usual notions of vectors, matrices, and tensors, without named indices.

Define \([n] = \{1, \ldots, n\}\). We can think of a size-\(n\) real vector \(v\) as a function from \([n]\) to \(\mathbb{R}\). We get the \(i\)th element of \(v\) by applying \(v\) to \(i\), but we normally write this as \(v_i\) (instead of \(v(i)\)).

Similarly, we can think of an \(m \times n\) real matrix as a function from \([m] \times [n]\) to \(\mathbb{R}\), and an \(l \times m \times n\) real tensor as a function from \([l] \times [m] \times [n]\) to \(\mathbb{R}\). In general, then, real tensors are functions from tuples of natural numbers to reals.

Named tensors

A named tuple looks like this: \[\{\ensuremath{\ensuremath{\mathsf{foo}}(1)}, \ensuremath{\ensuremath{\mathsf{bar}}(3)}\}.\] The order of the elements, called named indices, doesn’t matter: \[\{\ensuremath{\ensuremath{\mathsf{foo}}(1)}, \ensuremath{\ensuremath{\mathsf{bar}}(3)}\} = \{\ensuremath{\ensuremath{\mathsf{bar}}(3)}, \ensuremath{\ensuremath{\mathsf{foo}}(1)}\}.\] We use sans-serif font for names.

A named shape looks like this: \[\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times \ensuremath{\ensuremath{\mathsf{bar}}[3]}.\] Again, the order of the elements, called named index sets, doesn’t matter: \[\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times \ensuremath{\ensuremath{\mathsf{bar}}[3]} = \ensuremath{\ensuremath{\mathsf{bar}}[3]} \times \ensuremath{\ensuremath{\mathsf{foo}}[2]}.\] A named shape defines a set of possible named tuples: in this example, an index named \(\ensuremath{\mathsf{foo}}\) ranges from 1 to 2, and an index named \(\ensuremath{\mathsf{bar}}\) ranges from 1 to 3.

Then, a real named tensor is a function from a named shape to the real numbers. For example, here is a tensor with shape \(\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times \ensuremath{\ensuremath{\mathsf{bar}}[3]}\). \[A = \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3 & 1 & 4 \\ 1 & 5 & 9 \end{bmatrix}\end{array}.\]

We use uppercase italic letters for variables standing for named tensors. We don’t mind if you use another convention, but urge you not to use different styles for tensors and their elements. For example, if \(\mathbf{A}\) is a tensor, then an element of \(\mathbf{A}\) is written as \(\mathbf{A}_{\ensuremath{\ensuremath{\mathsf{foo}}(2)}, \ensuremath{\ensuremath{\mathsf{bar}}(3)}}\) – not \(A_{\ensuremath{\ensuremath{\mathsf{foo}}(2)},\ensuremath{\ensuremath{\mathsf{bar}}(3)}}\) or \(a_{\ensuremath{\ensuremath{\mathsf{foo}}(2)},\ensuremath{\ensuremath{\mathsf{bar}}(3)}}\).

Just as the set of all size-\(n\) real vectors is written \(\mathbb{R}^n\), and the set of all \(m\times n\) real matrices is often written \(\mathbb{R}^{m \times n}\) (which makes sense because one sometimes writes \(Y^X\) for the set of all functions from \(X\) to \(Y\)), we write \(\mathbb{R}^{\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times \ensuremath{\ensuremath{\mathsf{bar}}[3]}}\) for the set of all tensors with shape \(\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times \ensuremath{\ensuremath{\mathsf{bar}}[3]}\).

We access elements of \(A\) using subscripts: \(A_{\ensuremath{\ensuremath{\mathsf{foo}}(1)}, \ensuremath{\ensuremath{\mathsf{bar}}(3)}} = A_{\ensuremath{\ensuremath{\mathsf{bar}}(3)}, \ensuremath{\ensuremath{\mathsf{foo}}(1)}} = 4\). We also allow partial indexing: \[\begin{aligned} A_{\ensuremath{\ensuremath{\mathsf{foo}}(1)}} &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3 & 1 & 4 \end{bmatrix}\end{array} \\ A_{\ensuremath{\ensuremath{\mathsf{bar}}(3)}} &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{foo}}\\\begin{bmatrix} 4 & 9 \end{bmatrix}\end{array}.\end{aligned}\]

What are good choices for index names? We recommend meaningful words instead of single letters, and we recommend words that describe a whole rather than its parts. For example, a minibatch of sentences, each of which is a sequence of one-hot vectors, would be represented by a tensor with three indices, which we might name \(\ensuremath{\mathsf{batch}}\), \(\ensuremath{\mathsf{seq}}\), and \(\ensuremath{\mathsf{vocab}}\). Please see §3 for more examples.

Named tensor operations

Elementwise operations

Any function from scalars to scalars can be applied elementwise to a named tensor: \[\exp A = \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} \exp 3 & \exp 1 & \exp 4 \\ \exp 1 & \exp 5 & \exp 9 \end{bmatrix}\end{array}.\] More elementwise unary operations: \[\begin{array}{cl} kA & \text{scalar multiplication by $k$} \\ -A & \text{negation} \\ \exp A & \text{elementwise exponential function} \\ \tanh A & \text{hyperbolic tangent} \\ \sigma(A) & \text{logistic sigmoid} \\ \text{ReLU}(A) & \text{rectified linear unit} \end{array}\]

Any function or operator that takes two scalar arguments can be applied elementwise to two named tensors with the same shape. If \(A\) is as above and \[B = \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 2 & 7 & 1 \\ 8 & 2 & 8 \end{bmatrix}\end{array}\] then \[A + B = \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3+2 & 1+7 & 4+1 \\ 1+8 & 5+2 & 9+8 \end{bmatrix}\end{array}.\]

But things get more complicated when \(A\) and \(B\) don’t have the same shape. If \(A\) and \(B\) each have an index with the same name (and size), the two indices are aligned, as above. But if \(A\) has an index named \(\ensuremath{\mathsf{i}}\) and \(B\) doesn’t, then we do broadcasting, which means effectively that we replace \(B\) with a new tensor \(B'\) that contains a copy of \(B\) for every value of index \(\ensuremath{\mathsf{i}}\). \[\begin{aligned} A + 1 &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3+1 & 1+1 & 4+1 \\ 1+1 & 5+1 & 9+1 \end{bmatrix}\end{array} \\ A + B_{\ensuremath{\ensuremath{\mathsf{foo}}(1)}} &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3+2 & 1+7 & 4+1 \\ 1+2 & 5+7 & 9+1 \end{bmatrix}\end{array} \\ A + B_{\ensuremath{\ensuremath{\mathsf{bar}}(3)}} &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3+1 & 1+1 & 4+1 \\ 1+8 & 5+8 & 9+8 \end{bmatrix}\end{array}.\end{aligned}\] Similarly, if \(B\) has an index named \(\ensuremath{\mathsf{i}}\) and \(A\) doesn’t, then we effectively replace \(A\) with a new tensor \(A'\) that contains a copy of \(A\) for every value of index \(\ensuremath{\mathsf{i}}\). If you’ve programmed with NumPy or any of its derivatives, this should be unsurprising to you.

More elementwise binary operations: \[\begin{array}{cl} A+B & \text{addition} \\ A-B & \text{subtraction} \\ A\odot B & \text{elementwise (Hadamard) product} \\ A / B & \text{elementwise division} \\ \max \{A, B\} & \text{elementwise maximum} \\ \min \{A, B\} & \text{elementwise minimum} \end{array}\]

Reductions

The same rules for alignment and broadcasting apply to functions that take tensor as arguments or return tensors. The gory details are in §5.3, but we present the most important subcases here. The first is reductions, which are functions from vectors to scalars. Unlike with functions on scalars, we always have to specify which index these functions apply to, using a subscript. (This is equivalent to the axis argument in NumPy and dim in PyTorch.)

For example, using the same example tensor \(A\) from above, \[\begin{aligned} \sum\limits_{\ensuremath{\mathsf{foo}}} A &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3+1 & 1+5 & 4+9 \end{bmatrix}\end{array} \\ \sum\limits_{\ensuremath{\mathsf{bar}}} A &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{foo}}\\\begin{bmatrix} 3+1+4 & 1+5+9 \end{bmatrix}\end{array}.\end{aligned}\] More reductions: If \(A\) has shape \(\ensuremath{\ensuremath{\mathsf{foo}}[I]} \times \ldots\), then \[\begin{aligned} \sum\limits_{\ensuremath{\mathsf{foo}}} A &= \sum_{i \in I} A_{\ensuremath{\ensuremath{\mathsf{foo}}(i)}} = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}4 & 6 & 13\end{bmatrix}\end{array} \\ \mathop{\mathrm{norm}}\limits_{\ensuremath{\mathsf{foo}}} A &= \sqrt{\sum\limits_{\ensuremath{\mathsf{foo}}} A^2} = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}\sqrt{10} & \sqrt{26} & \sqrt{97}\end{bmatrix}\end{array} \\ \mathop{\mathrm{min}}\limits_{\ensuremath{\mathsf{foo}}} A &= \min_{i \in I} A_{\ensuremath{\ensuremath{\mathsf{foo}}(i)}} = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}1 & 1 & 4\end{bmatrix}\end{array} \\ \mathop{\mathrm{max}}\limits_{\ensuremath{\mathsf{foo}}} A &= \max_{i \in I} A_{\ensuremath{\ensuremath{\mathsf{foo}}(i)}} = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}3 & 5 & 9\end{bmatrix}\end{array} \\ \mathop{\mathrm{mean}}\limits_{\ensuremath{\mathsf{foo}}} A &= \frac{1}{|I|} A = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}2 & 3 & 6.5\end{bmatrix}\end{array} \\ \mathop{\mathrm{var}}\limits_{\ensuremath{\mathsf{foo}}} A &= \frac1{|I|} \sum\limits_{\ensuremath{\mathsf{i}}} (A - \mathop{\mathrm{mean}}\limits_{\ensuremath{\mathsf{foo}}} A)^2 = \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix}1 & 4 & 6.25\end{bmatrix}\end{array}\end{aligned}\] (Note that \(\max\) and \(\min\) are overloaded; with multiple arguments and no subscript, they are elementwise, and with a single argument and a subscript, they are reductions.)

You can also write multiple names to perform the reduction over multiple indices at once.

Contraction

The vector dot product (inner product) is a function from two vectors to a scalar, which generalizes to named tensors to give the ubiquitous contraction operator, which performs elementwise multiplication, then sums along an index. It can be used, for example, for matrix multiplication: \[\begin{aligned} C &= \ensuremath{\mathsf{bar}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix} 1 & -1 \\ 2 & -2 \\ 3 & -3 \end{bmatrix}\end{array} \\ A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{bar}}}} C &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix} 17 & -17 \\ 53 & -53 \end{bmatrix}\end{array}\end{aligned}\] However, note that (like vector dot-product, but unlike matrix multiplication) this operator is commutative, but not associative! Specifically, if \[\begin{aligned} A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{foo}}[m]}} \\ B &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{foo}}[m]} \times\ensuremath{\ensuremath{\mathsf{bar}}[n]}} \\ C &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{foo}}[m]} \times\ensuremath{\ensuremath{\mathsf{bar}}[n]}}\end{aligned}\] then \((A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{foo}}}} B) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{bar}}}} C\) and \(A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{foo}}}} (B \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{bar}}}} C)\) don’t even have the same shape.

Vectors to vectors

A very common example of a function from vectors to vectors is the softmax: \[\mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{foo}}} A = \frac{\exp A}{\sum\limits_{\ensuremath{\mathsf{foo}}} \exp A} \approx \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 0.731 & 0.002 & 0.953 \\ 0.269 & 0.998 & 0.047 \end{bmatrix}\end{array}.\]

And it’s also very handy to have a function that renames an index: \[\left[A\right]_{\ensuremath{\mathsf{bar}}\rightarrow\ensuremath{\mathsf{baz}}} = \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix} 3 & 1 & 4 \\ 1 & 5 & 9 \end{bmatrix}\end{array}\]

Concatenation combines two vectors into one: \[\begin{aligned} A \mathbin{\mathop{\oplus}\limits_{\ensuremath{\mathsf{foo}}}} B &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3 & 1 & 4 \\ 1 & 5 & 9 \\ 2 & 7 & 1 \\ 8 & 2 & 8 \end{bmatrix}\end{array} \\ A \mathbin{\mathop{\oplus}\limits_{\ensuremath{\mathsf{bar}}}} B &= \ensuremath{\mathsf{foo}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{bar}}\\\begin{bmatrix} 3 & 1 & 4 & 2 & 7 & 1 \\ 1 & 5 & 9 & 8 & 2 & 8 \end{bmatrix}\end{array}\end{aligned}\]

Matrices

Finally, we briefly consider functions on matrices, for which you have to give two index names (and the order in general matters). Let \(A\) be a named tensor with shape \(\{\ensuremath{\ensuremath{\mathsf{foo}}[2]} \times\ensuremath{\ensuremath{\mathsf{bar}}[2]} \times\ensuremath{\ensuremath{\mathsf{baz}}[2\}]}\): \[\begin{aligned} A_{\ensuremath{\ensuremath{\mathsf{foo}}(1)}} &= \ensuremath{\mathsf{bar}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix}\end{array} \\ A_{\ensuremath{\ensuremath{\mathsf{foo}}(2)}} &= \ensuremath{\mathsf{bar}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix} 5 & 6 \\ 7 & 8 \end{bmatrix}\end{array} \\ \mathop{\mathrm{det}}\limits_{\ensuremath{\mathsf{bar,baz}}} A &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{foo}}\\\begin{bmatrix}\det \begin{bmatrix} 1 & 2 \\ 3 & 4 \end{bmatrix} & \det \begin{bmatrix} 5 & 6 \\ 7 & 8 \end{bmatrix}\end{bmatrix}\end{array} \\ \mathop{\mathrm{det}}\limits_{\ensuremath{\mathsf{baz,bar}}} A &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{foo}}\\\begin{bmatrix}\det \begin{bmatrix} 1 & 3 \\ 2 & 4 \end{bmatrix} & \det \begin{bmatrix} 5 & 7 \\ 6 & 8 \end{bmatrix}\end{bmatrix}\end{array} \\ \mathop{\mathrm{det}}\limits_{\ensuremath{\mathsf{foo,bar}}} A &= \ensuremath{\mathsf{}}\begin{array}[b]{@{}c@{}}\ensuremath{\mathsf{baz}}\\\begin{bmatrix}\det \begin{bmatrix} 1 & 3 \\ 5 & 7 \end{bmatrix} & \det \begin{bmatrix} 2 & 4 \\ 6 & 8 \end{bmatrix}\end{bmatrix}\end{array}\end{aligned}\] For matrix inverses, there’s no easy way to put a subscript under \(\mathord\cdot^{-1}\), so we recommend writing \(\mathop{\mathrm{inv}}\limits_{\ensuremath{\mathsf{foo,bar}}}\).

Examples

In this section we give a series of examples illustrating how to use named tensors in various situations, mostly related to machine learning.

As further proof of concept, in separate pages we have written out the full models for Transformer (https://namedtensor.github.io/transformer.html) and LeNet (https://namedtensor.github.io/convnet.html).

Attention

Let \(d_k\) and \(d_v\) be positive integers, and let \(n\) and \(n'\) be the input and output sequence length. Define a function \[\begin{aligned} \text{Att} &\colon \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq'}}[n']} \times\ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \times \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \times \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \rightarrow \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq'}}[n']} \times\ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \\ \text{Att}(Q,K,V) &= \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{time}}} \left( \frac{Q \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{key}}}} K}{\sqrt{d_k}} \right) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{seq}}}} V\end{aligned}\]

In self-attention, \(Q\), \(K\), and \(V\) are all computed from the same sequence. Let \(d_{\text{model}}\) be a positive integer. The parameters are: \[\begin{aligned} W^Q &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \\ W^K &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \\ W^V &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \\ W^O &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{val}}[d_v]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}}\end{aligned}\] Then define \[\begin{aligned} \text{SelfAtt} &\colon \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}} \rightarrow \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}} \\ \text{SelfAtt}(X; W^Q, W^K, W^V, W^O) &= W^O \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{val}}}} \left[\text{Att}(Q, K, V)\right]_{\ensuremath{\mathsf{seq'}}\rightarrow\ensuremath{\mathsf{seq}}}\end{aligned}\] where \[\begin{aligned} Q &= W^Q \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} \left[X\right]_{\ensuremath{\mathsf{seq}}\rightarrow\ensuremath{\mathsf{seq'}}} \\ K &= W^K \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} X \\ V &= W^V \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} X.\end{aligned}\]

To change this to multi-head self-attention with \(h\) attention heads, simply redefine \[\begin{aligned} W^Q &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{head}}[h]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \\ W^K &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{head}}[h]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \\ W^V &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{head}}[h]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]} \times \ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \\ W^O &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{head}}[h]} \times \ensuremath{\ensuremath{\mathsf{val}}[d_v]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}}\end{aligned}\] and define \[\begin{aligned} \text{MultiSelfAtt} &\colon \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}} \rightarrow \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{emb}}[d_{\text{model}}]}} \\ \text{MultiSelfAtt}(X; W^Q, W^K, W^V, W^O) &= \sum\limits_{\ensuremath{\mathsf{head}}} \text{SelfAtt}(X; W^Q, W^K, W^V, W^O).\end{aligned}\]

Finally, when the Transformer is used for generation, it is necessary to have an additional mask to ensure the model does not look at future words. This can be included in the attention definition with clear names.

\[\begin{aligned} \text{Attention}(Q, K, V) &= \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{seq}}} \left( \frac{Q \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{key}}}} K}{\sqrt{d_k}} + M \right) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{seq}}}} V \\ M & \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times \ensuremath{\ensuremath{\mathsf{seq'}}[n]}} \\ M_{\ensuremath{\ensuremath{\mathsf{seq}}(i)}, \ensuremath{\ensuremath{\mathsf{seq'}}(j)}} & = \begin{cases}0 & i \leq j\\ -\infty & \text{otherwise.} \end{cases}\end{aligned}\]

Recurrent neural networks

As a second example, let’s define a simple (Elman) RNN. Let \(d\) be a positive integer.

\[\begin{aligned} x^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]}} & t &= 1, \ldots, n \\ h^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} & t &= 0, \ldots, n\\ A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]} \times \ensuremath{\ensuremath{\mathsf{state'}}[d]}} \\ B &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]} \times \ensuremath{\ensuremath{\mathsf{state'}}[d]}} \\ c &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state'}}[d]}} \\ h^{(t+1)} &= \left[\tanh\left( A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{state}}}} h^{(t)} + B \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} x^{(t)} + c \right)\right]_{\ensuremath{\mathsf{state'}}\rightarrow\ensuremath{\mathsf{state}}}\end{aligned}\]

The renaming is necessary because our notation doesn’t provide a one-step way to apply a linear transformation (\(A\)) to one index and put the result in the same index. For possible solutions, see §6.2.

Fully-connected layers

Fully-connected layers are bit more verbose, but make more explicit which parameters connect which layers.

\[\begin{aligned} V &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{output}}[o]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}} & c\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{output}}[o]}} \\ W &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{hidden}}[h]} \times \ensuremath{\ensuremath{\mathsf{in}}[i]}}} & b \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{hidden}}[h]}} \\ X &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{in}}[i]}}} \\ Y &= \sigma \left( W \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{in}}}} X + b \right) \\ Z &= \sigma \left( V \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{hidden}}}} Y + c \right) \end{aligned}\]

Normalization layers

These three functions are often informally described using the same equation, but they each correspond to very different functions. They differ by which axes are normalized.

Batch normalization

\[\begin{aligned} X &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}}\\ \gamma, \beta &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]}}} \\ \text{batchnorm}(X; \gamma, \beta) &= \frac{X - \mathop{\mathrm{mean}}\limits_{\ensuremath{\mathsf{batch}}}(X)}{\sqrt{\mathop{\mathrm{var}}\limits_{\ensuremath{\mathsf{batch}}}(X)} + \epsilon} \odot \gamma + \beta\end{aligned}\]

Instance normalization

\[\begin{aligned} X &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}}\\ \gamma, \beta &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{hidden}}[h]}}} \\ \text{instancenorm}(X; \gamma, \beta) &= \frac{X - \mathop{\mathrm{\text{mean}}}\limits_{\ensuremath{\mathsf{hidden}}}(X)}{\sqrt{\mathop{\mathrm{\text{var}}}\limits_{\ensuremath{\mathsf{hidden}}}(X)} + \epsilon} \odot \gamma + \beta\end{aligned}\]

Layer normalization

\[\begin{aligned} X &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}} \\ \gamma, \beta &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}} \\ \text{layernorm}(X; \gamma, \beta) &= \frac{X - \mathop{\mathrm{\text{mean}}}\limits_{\ensuremath{\mathsf{hidden,channels}}}(X)}{\sqrt{\mathop{\mathrm{\text{var}}}\limits_{\ensuremath{\mathsf{hidden, channels}}}(X)} + \epsilon} \odot \gamma + \beta \end{aligned}\]

Continuous bag of words

A continuous bag-of-words model classifies by summing up the embeddings of a sequence of words \(X\) and then projecting them to the space of classes.

\[\begin{aligned} X &\in \{0, 1\}^{{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times \ensuremath{\ensuremath{\mathsf{vocab}}[v]}}} & \sum\limits_{\ensuremath{\mathsf{vocab}}} X &= 1\\ E &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{vocab}}[v]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}}\\ W &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{class}}[c]} \times \ensuremath{\ensuremath{\mathsf{hidden}}[h]}}}\\ \text{cbow}(X; E, W) &= \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{class}}} (W \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{hidden}}}} E \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{vocab}}}} X)\end{aligned}\] Here, the two contractions can be done in either order, so we leave the parentheses off.

Discrete random variables

Named indices are very helpful for working with discrete random variables, because each random variable can be represented by an index with the same name. For instance, if \(\ensuremath{\mathsf{A}}\) and \(\ensuremath{\mathsf{B}}\) are random variables, we can treat \(p(\ensuremath{\mathsf{B}} \mid \ensuremath{\mathsf{A}})\) and \(p(\ensuremath{\mathsf{A}})\) as tensors: \[\begin{aligned} p(\ensuremath{\mathsf{B}} \mid \ensuremath{\mathsf{A}}) &\in [0, 1]^{\ensuremath{\ensuremath{\mathsf{A}}[a]} \times \ensuremath{\ensuremath{\mathsf{B}}[b]}} & \sum\limits_{\ensuremath{\mathsf{B}}} p(\ensuremath{\mathsf{B}}\mid \ensuremath{\mathsf{A}}) &= 1 \\ p(\ensuremath{\mathsf{A}}) &\in [0, 1]^{\ensuremath{\ensuremath{\mathsf{A}}[a]}} & \sum\limits_{\ensuremath{\mathsf{A}}} p(\ensuremath{\mathsf{A}}) &= 1\end{aligned}\] Then Bayes’ rule is just: \[\begin{aligned} p(\ensuremath{\mathsf{A}} \mid \ensuremath{\mathsf{B}}) &= \frac{p(\ensuremath{\mathsf{B}} \mid \ensuremath{\mathsf{A}}) \odot p(\ensuremath{\mathsf{A}})}{p(\ensuremath{\mathsf{B}} \mid \ensuremath{\mathsf{A}}) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{A}}}} p(\ensuremath{\mathsf{A}})}.\end{aligned}\]

Sudoku ILP

Sudoku puzzles can be represented as binary tiled tensors. Given a grid we can check that it is valid by converting it to a grid of grids. Constraints then ensure that there is one digit per row, per column and per sub-box.

\[\begin{aligned} X &\in \{0, 1\}^{\ensuremath{\ensuremath{\mathsf{height}}[9]} \times \ensuremath{\ensuremath{\mathsf{width}}[9]} \times \ensuremath{\ensuremath{\mathsf{assign}}[9]}} \\ \text{check}(X) &= \left(\sum\limits_{\ensuremath{\mathsf{assign}}} Y = \sum\limits_{\ensuremath{\mathsf{Height, height}}} Y = \sum\limits_{\ensuremath{\mathsf{Width, width}}} Y = \sum\limits_{\ensuremath{\mathsf{height, width}}} Y = 1 \right) \\ Y &\in \{0, 1\}^{\ensuremath{\ensuremath{\mathsf{Height}}[3]} \times \ensuremath{\ensuremath{\mathsf{Width}}[3]} \times \ensuremath{\ensuremath{\mathsf{height}}[3]} \times \ensuremath{\ensuremath{\mathsf{width}}[3]} \times \ensuremath{\ensuremath{\mathsf{assign}}[9]}} \\ Y_{\ensuremath{\ensuremath{\mathsf{Height}}(r)}, \ensuremath{\ensuremath{\mathsf{height}}(r')}, \ensuremath{\ensuremath{\mathsf{Width}}(c)}, \ensuremath{\ensuremath{\mathsf{width}}(c')}} &= X_{\ensuremath{\ensuremath{\mathsf{height}}(r\times 3 + r'-1)}, \ensuremath{\ensuremath{\mathsf{width}}(c\times 3 + c'-1)}}\end{aligned}\]

Max pooling

Max pooling used in image recognition takes a similar form as the Sudoku example.

\[\begin{aligned} X &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{height}}[h]} \times \ensuremath{\ensuremath{\mathsf{width}}[w]}} \\ \text{maxpool2d}(X, kh, kw) &= \mathop{\mathrm{max}}\limits_{\ensuremath{\mathsf{kh, kw}}} U \\ U &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{height}}[h / kh]} \times \ensuremath{\ensuremath{\mathsf{width}}[w / kw]} \times \ensuremath{\ensuremath{\mathsf{kh}}[kh]} \times \ensuremath{\ensuremath{\mathsf{kw}}[kw]}}} \\ U_{\ensuremath{\ensuremath{\mathsf{height}}(i)}, \ensuremath{\ensuremath{\mathsf{width}}(j)}, \ensuremath{\ensuremath{\mathsf{kh}}(di)}, \ensuremath{\ensuremath{\mathsf{kw}}(dj)}} & = X_{\ensuremath{\ensuremath{\mathsf{height}}(i \times kh + di -1)}, \ensuremath{\ensuremath{\mathsf{width}}(j \times kw + dj -1)}} \end{aligned}\]

1D convolution

1D convolution can be easily written by unrolling a tensor and then applying a standard dot product.

\[\begin{aligned} X &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{seq}}[n]}}} \\ W &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{out\_channels}}[c']} \times \ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{kw}}[k]}}} \\ \text{conv1d}(X, W) &= W \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{channels, kw}}}} U \\ U &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{channels}}[c]} \times \ensuremath{\ensuremath{\mathsf{seq}}[n-k +1]} \times \ensuremath{\ensuremath{\mathsf{kw}}[k]}}} \\ U_{\ensuremath{\ensuremath{\mathsf{seq}}(i)}, \ensuremath{\ensuremath{\mathsf{kw}}(j)}} & = X_{\ensuremath{\ensuremath{\mathsf{seq}}(i+j - 1)}} \end{aligned}\]

\(K\)-means clustering

The following equations define one step of \(k\)-means clustering. Given a set of points \(X\) and an initial set of cluster centers \(C\), \[\begin{aligned} X &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{space}}[k]}} \\ C &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{clusters}}[c]} \times \ensuremath{\ensuremath{\mathsf{space}}[k]}}\end{aligned}\] we compute cluster assignments \[\begin{aligned} % Q &\in \{0, 1\}^{\nset{batch}{b} \times\nset{clusters}{c}} \\ Q &= \mathop{\mathrm{argmin}}\limits_{\ensuremath{\mathsf{clusters}}} \mathop{\mathrm{norm}}\limits_{\ensuremath{\mathsf{space}}}(C-X) \\ &= \lim_{\alpha \rightarrow -\infty} \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{clusters}}} \left(\alpha \mathop{\mathrm{norm}}\limits_{\ensuremath{\mathsf{space}}}(C-X)\right)\end{aligned}\] then we recompute the cluster centers: \[C \leftarrow \sum\limits_{\ensuremath{\mathsf{batch}}} \frac{Q \odot X}{Q}.\]

Beam search

Beam search is a commonly used approach for approximate discrete search. Here \(H\) is the score of each element in the beam, \(S\) is the state of each element in the beam, and \(f\) is an update function that returns the score of each state transition. Beam step returns the new \(H\) tensor.

\[\begin{aligned} H &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{beam}}[k]}}} \\ S &\in \{0, 1\}^{{\ensuremath{\ensuremath{\mathsf{batch}}[b]} \times \ensuremath{\ensuremath{\mathsf{beam}}[k]} \times \ensuremath{\ensuremath{\mathsf{state}}[s]}}} & \sum\limits_{\ensuremath{\mathsf{state}}} S &= 1\\ f &\colon \{0, 1\}^{{\ensuremath{\ensuremath{\mathsf{state}}[s]}}} \rightarrow \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{state'}}[s]}}} \\ \text{beamstep}(H, S) &= \mathop{\mathrm{maxk}}\limits_{\ensuremath{\mathsf{beam, state'}}} \left( \mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{state'}}}(f(S)) \odot H \right)\end{aligned}\]

Multivariate normal distribution

In our notation, the application of a bilinear form is more verbose than the standard notation (\((X-\mu)^\top \Sigma^{-1} (X-\mu)\)), but also makes it look more like a function of two arguments (and would generalize to three or more arguments).

\[\begin{aligned} X &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{batch}}[{b}]} \times \ensuremath{\ensuremath{\mathsf{d}}[{k}]}} \\ \mu &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{d}}[{k}]}}} \\ \Sigma & \in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{d1}}[{k}]} \times \ensuremath{\ensuremath{\mathsf{d2}}[{k}]}}} \\ {\cal N}(X; \mu, \Sigma) &= \frac{\displaystyle \exp\left(-\frac{1}{2} \left(\mathop{\mathrm{inv}}\limits_{\ensuremath{\mathsf{d1, d2}}}(\Sigma) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{d1}}}} \left[X - \mu\right]_{\ensuremath{\mathsf{d}}\rightarrow\ensuremath{\mathsf{d1}}} \right) \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{d2}}}} \left[X - \mu\right]_{\ensuremath{\mathsf{d}}\rightarrow\ensuremath{\mathsf{d2}}} \right)}{\sqrt{(2 \pi)^k \mathop{\mathrm{det}}\limits_{\ensuremath{\mathsf{d1, d2}}}(\Sigma)}}\end{aligned}\]

Formal Definitions

Named tuples

A named index is a pair, written \(\ensuremath{\ensuremath{\mathsf{i}}(x)}\), where \(\ensuremath{\mathsf{i}}\) is a name and \(x\) is usually a positive integer. We write both names and variables ranging over names using sans-serif font.

A named tuple is a set of named indices \(\{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}(x_1)}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}(x_r)}\}\), where \(\ensuremath{\mathsf{i_\text{$1$}}}, \ldots \ensuremath{\mathsf{i_\text{$r$}}}\) are pairwise distinct names.

If \(\ensuremath{\mathsf{i}}\) is a name and \(X\) is a set, define the named index set \(\ensuremath{\ensuremath{\mathsf{i}}[X]}\) to be the set \(\{\ensuremath{\ensuremath{\mathsf{i}}(x)} \mid x \in X\}\). We deal with named index sets of the form \(\ensuremath{\ensuremath{\mathsf{i}}[[n]]}\) (\({}=\ensuremath{\ensuremath{\mathsf{i}}[\{1, \ldots, n\}]}\)) so frequently that we abbreviate this as \(\ensuremath{\ensuremath{\mathsf{i}}[n]}\).

A named shape is a set of named index sets \(\{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}[X_1]}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}[X_r]}\}\), where \(\ensuremath{\mathsf{i_\text{$1$}}}, \ldots \ensuremath{\mathsf{i_\text{$r$}}}\) are pairwise distinct names. A named shape defines a set of named tuples: \[\tupleshape \{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}[X_1]}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}[X_r]}\} = \left\{\{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}(x_1)}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}(x_r)}\} \mid x_1 \in X_1, \ldots, x_r \in X_r\right\},\] and for this reason we often write a shape with the alternative notation \(\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}[X_1]} \times \cdots \times \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}[X_r]}\), because it’s like an “unordered Cartesian product” of the index sets.

We say two shapes \(\mathcal{S}\) and \(\mathcal{T}\) are compatible if whenever \(\ensuremath{\ensuremath{\mathsf{i}}(X)} \in \mathcal{S}\) and \(\ensuremath{\ensuremath{\mathsf{i}}(Y)} \in \mathcal{T}\), then \(X = Y\). We say that \(\mathcal{S}\) and \(\mathcal{T}\) are orthogonal if there is no \(\ensuremath{\mathsf{i}}\) such that \(\ensuremath{\ensuremath{\mathsf{i}}(X)} \in \mathcal{S}\) and \(\ensuremath{\ensuremath{\mathsf{i}}(Y)} \in \mathcal{T}\) for any \(X\), \(Y\).

If \(t \in \tupleshape \mathcal{T}\) and \(\mathcal{S} \subseteq \mathcal{T}\), then we write \(\left.t\right|_{\mathcal{S}}\) for the unique tuple in \(\tupleshape{\mathcal{S}}\) such that \(\left.t\right|_{\mathcal{S}} \subseteq t\).

Named tensors

Let \(F\) be a field and let \(\mathcal{T}\) be a set of named tuples. Then a named tensor over \(F\) with shape \(\mathcal{T}\) is a mapping from \(\mathcal{T}\) to \(F\). We write the set of all named tensors with shape \(\mathcal{T}\) as \(F^{\mathcal{T}}\).

We don’t make any distinction between a scalar (an element of \(F\)) and a named tensor with empty shape (an element of \(F^\emptyset\)).

If \(A \in F^{\mathcal{T}}\), then we access an element of \(A\) by applying it to a named tuple \(t \in \tupleshape\mathcal{T}\); but we write this using the usual subscript notation: \(A_t\) rather than \(A(t)\). To avoid clutter, in place of \(A_{\{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}(x_1)}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}(x_r)}\}}\), we usually write \(A_{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}(x_1)}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}(x_r)}}\). When a named tensor is an expression like \((A+B)\), we surround it with square brackets like this: \([A+B]_{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}(x_1)}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$r$}}}(x_r)}}\).

We also allow partial indices. If \(A\) is a tensor with shape \(\mathcal{T}\) and \(s \in \tupleshape \mathcal{S}\) where \(\mathcal{S} \subseteq \mathcal{T}\), then we define \(A_s\) to be the named tensor with shape \(\mathcal{T} \setminus \mathcal{S}\) such that, for any \(t \in \tupleshape (\mathcal{T} \setminus \mathcal{S})\), \[\begin{aligned} \left[A_s\right]_t &= A_{s \cup t}.\end{aligned}\] (For the edge case \(\mathcal{S} = \mathcal{U}\) and \(\mathcal{T} = \emptyset\), our definitions for indexing and partial indexing coincide: one gives a scalar and the other gives a tensor with empty shape, but we don’t distinguish between the two.)

Extending functions to named tensors

In §2, we described several classes of functions that can be extended to named tensors. Here, we define how to do this for general functions.

Let \(f \colon F^{\mathcal{S}} \rightarrow G^{\mathcal{T}}\) be a function from tensors to tensors. For any shape \(\mathcal{U}\) orthogonal to both \(\mathcal{S}\) and \(\mathcal{T}\), we can extend \(f\) to: \[\begin{aligned} f &: F^{\mathcal{S} \cup \mathcal{U}} \rightarrow G^{\mathcal{T} \cup \mathcal{U}} \\ [f(A)]_u &= f(A_u) \qquad \text{for all $u \in \tupleshape\mathcal{U}$.}\end{aligned}\]

If \(f\) is a multary function, we can extend its arguments to larger shapes, and we don’t have to extend all the arguments with the same names. We consider just the case of two arguments; three or more arguments are analogous. Let \(f \colon F^{\mathcal{S}} \times G^{\mathcal{T}} \rightarrow H^{\mathcal{U}}\) be a binary function from tensors to tensors. For any shapes \(\mathcal{S'}\) and \(\mathcal{T'}\) that are compatible with each other and orthogonal to \(\mathcal{S}\) and \(\mathcal{T}\), respectively, and \(\mathcal{U'} = \mathcal{S'} \cup \mathcal{T'}\) is orthogonal to \(\mathcal{U}\), we can extend \(f\) to: \[\begin{aligned} f &: F^{\mathcal{S} \cup \mathcal{S'}} \times G^{\mathcal{T} \cup \mathcal{T'}} \rightarrow H^{\mathcal{U} \cup \mathcal{U'}} \\ [f(A,B)]_u &= f\left(A_{\left.u\right|_{\mathcal{S'}}},B_{\left.u\right|_{\mathcal{T'}}}\right) \qquad \text{for all $u \in \tupleshape\mathcal{U'}$.}\end{aligned}\]

All of the tensor operations described in §2.2 can be defined in this way. For example, the contraction operator extends the following “named dot-product”: \[\begin{aligned} \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} &: F^{\ensuremath{\ensuremath{\mathsf{i}}[n]}} \times F^{\ensuremath{\ensuremath{\mathsf{i}}[n]}} \rightarrow F \\ A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} B &= \sum_{i=1}^n A_{\ensuremath{\ensuremath{\mathsf{i}}(i)}} B_{\ensuremath{\ensuremath{\mathsf{i}}(i)}}.\end{aligned}\]

Extensions

Index types

We have defined a named index set as a pair \(\ensuremath{\ensuremath{\mathsf{i}}[X]}\), where \(\ensuremath{\mathsf{i}}\) is a name and \(X\) is a set, usually \([n]\) for some \(n\). In this section, we consider some other possibilities for \(X\).

Non-integral types

The sets \(X\) don’t have to contain integers. For example, if \(V\) is the vocabulary of a natural language (\(V = \{ \ensuremath{\mathsf{cat}}, \ensuremath{\mathsf{dog}}, \ldots \}\)), we could define a matrix of word embeddings: \[\begin{aligned} E &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{vocab}}[V]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d]}}.\end{aligned}\]

Integers with units

If \(\ensuremath{\mathsf{u}}\) is a symbol and \(n > 0\), define \([n]\ensuremath{\mathsf{u}} = \{1\ensuremath{\mathsf{u}}, 2\ensuremath{\mathsf{u}}, \ldots, n\ensuremath{\mathsf{u}}\}\). You could think of \(\ensuremath{\mathsf{u}}\) as analogous to a physical unit, like kilograms. The elements of \([n]\ensuremath{\mathsf{u}}\) can be added and subtracted like integers (\(a\ensuremath{\mathsf{u}} + b\ensuremath{\mathsf{u}} = (a+b)\ensuremath{\mathsf{u}}\)) or multiplied by unitless integers (\(c \cdot a\ensuremath{\mathsf{u}} = (c \cdot a) \ensuremath{\mathsf{u}}\)), but numbers with different units are different (\(a \ensuremath{\mathsf{u}} \neq a \ensuremath{\mathsf{v}}\)).

Then the set \([n]\ensuremath{\mathsf{u}}\) could be used as an index set, which would prevent the index from being aligned with another index that uses different units. For example, if we want to define a tensor representing an image, we might write \[A \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{height}}[[h]\ensuremath{\mathsf{pixels}}]} \times \ensuremath{\ensuremath{\mathsf{width}}[[w]\ensuremath{\mathsf{pixels}}]}}.\] If we have another tensor representing a go board, we might write \[B \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{height}}[[n]\ensuremath{\mathsf{points}}]} \times \ensuremath{\ensuremath{\mathsf{width}}[[n]\ensuremath{\mathsf{points}}]}},\] and even if it happens that \(h = w = n\), it would be incorrect to write \(A+B\) because the units do not match.

Tuples of integers

An index set could also be \([m] \times [n]\), which would be a way of sneaking ordered indices into named tensors, useful for matrix operations. For example, instead of defining an \(\text{inv}\) operator that takes two subscripts, we could write \[\begin{aligned} A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{d}}[{m\times n}]}} = \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{d}}[{[m]\times [n]}]}} \\ B &= \mathop{\mathrm{inv}}\limits_{\ensuremath{\mathsf{d}}} A.\end{aligned}\] We could also define an operator \(\circ\) for matrix-matrix and matrix-vector multiplication: \[\begin{aligned} c &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{d}}[n]}} \\ D &= A \mathbin{\mathop{\circ}\limits_{\ensuremath{\mathsf{d}}}} B \mathbin{\mathop{\circ}\limits_{\ensuremath{\mathsf{d}}}} c.\end{aligned}\]

Duality

In applied linear algebra, we distinguish between column and row vectors; in pure linear algebra, vector spaces and dual vector spaces; in tensor algebra, contravariant and covariant indices; in quantum mechanics, bras and kets. Do we need something like this?

In §3.2 we saw that defining an RNN requires renaming of indices, because a linear transformation must map one index to another index; if we want to map an index to itself, we need to use renaming.

In this section, we describe three possible solutions to this problem, and welcome comments about which (if any) would be best.

Contracting two names

We define a version of the contraction operator that can contract two indices with different names (and the same size): \[\begin{aligned} \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}|\ensuremath{\mathsf{j}}}} &: F^{\ensuremath{\ensuremath{\mathsf{i}}[n]}} \times F^{\ensuremath{\ensuremath{\mathsf{j}}[n]}} \rightarrow F \\ A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}|\ensuremath{\mathsf{j}}}} B &= \sum_{k=1}^n A_{\ensuremath{\ensuremath{\mathsf{i}}(k)}} B_{\ensuremath{\ensuremath{\mathsf{j}}(k)}}.\end{aligned}\]

For example, the RNN would look like this. \[\begin{aligned} x^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]}} \\ h^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]} \times \ensuremath{\ensuremath{\mathsf{state'}}[d]}} \\ B &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]} \times \ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ c &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ h^{(t+1)} &= \tanh\left( A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{state'}}|\ensuremath{\mathsf{state}}}} h^{(t)} + B \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} x^{(t)} + c \right)\end{aligned}\]

Starred index names

If \(\ensuremath{\mathsf{i}}\) is a name, we also allow a tensor to have an index \(\ensuremath{\mathsf{i*}}\) (alternatively: superscript \(\ensuremath{\mathsf{i}}\)). Multiplication contracts starred indices in the left operand with non-starred indices in the right operand. \[\begin{aligned} x^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]}} \\ h^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state*}}[d]} \times \ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ B &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb*}}[d]} \times \ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ c &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ h^{(t+1)} &= \tanh\left( A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{state}}}} h^{(t)} + B \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{emb}}}} x^{(t)} + c \right) \end{aligned}\] The contraction operator can be defined as: \[\begin{aligned} \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} &: F^{\ensuremath{\ensuremath{\mathsf{i*}}[n]}} \times F^{\ensuremath{\ensuremath{\mathsf{i}}[n]}} \rightarrow F \\ A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} B &= \sum_{i=1}^n A_{\ensuremath{\ensuremath{\mathsf{i*}}(i)}} B_{\ensuremath{\ensuremath{\mathsf{i}}(i)}}.\end{aligned}\]

There are a few variants of this idea that have been floated:

  1. \(\mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{}}}}\) (no subscript) contracts every starred index in its left operand with every corresponding unstarred index in its right operand. Rejected.

  2. \(\mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}}\) contracts \(\ensuremath{\mathsf{i}}\) with \(\ensuremath{\mathsf{i}}\), and we need another notation like \(\mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i(*)}}}}\) or \(\mathbin{\mathop{\times}\limits_{\ensuremath{\mathsf{i}}}}\) for contracting \(\ensuremath{\mathsf{i*}}\) with \(\ensuremath{\mathsf{i}}\).

  3. \(\mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}}\) always contracts \(\ensuremath{\mathsf{i*}}\) with \(\ensuremath{\mathsf{i}}\); there’s no way to contract \(\ensuremath{\mathsf{i}}\) with \(\ensuremath{\mathsf{i}}\).

Named and numbered indices

We allow indices to have names that are natural numbers \(1, 2, \ldots\), and we define “numbering” and “naming” operators:

\(A_{\ensuremath{\mathsf{i}}}\) rename index \(\ensuremath{\mathsf{i}}\) to 1
\(A_{\ensuremath{\mathsf{i}},\ensuremath{\mathsf{j}}}\) rename index \(\ensuremath{\mathsf{i}}\) to 1 and \(\ensuremath{\mathsf{j}}\) to 2
\(A_{\rightarrow\ensuremath{\mathsf{i}}}\) rename index 1 to \(\ensuremath{\mathsf{i}}\)
\(A_{\rightarrow\ensuremath{\mathsf{i}},\ensuremath{\mathsf{j}}}\) rename index 1 to \(\ensuremath{\mathsf{i}}\) and 2 to \(\ensuremath{\mathsf{j}}\)

The numbering operators are only defined on tensors that have no numbered indices.

Then we adopt the convention that standard vector/matrix operations operate on the numbered indices. For example, vector dot-product always uses index 1 of both its operands, so that we can write \[C = A_{\ensuremath{\mathsf{i}}} \cdot B_{\ensuremath{\mathsf{i}}}\] equivalent to \(C = A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} B\).

Previously, we had to define a new version of every operation; most of the time, it looked similar to the standard version (e.g., \(\max\) vs \(\max_{\ensuremath{\mathsf{i}}}\)), but occasionally it looked quite different (e.g., matrix inversion). With numbered indices, we can use standard notation for everything. (This also suggests a clean way to integrate code that uses named tensors with code that uses ordinary tensors.)

We also get the renaming operation for free: \(A_{\ensuremath{\mathsf{i}}\rightarrow\ensuremath{\mathsf{j}}} = [A_{\ensuremath{\mathsf{i}}}]_{\rightarrow\ensuremath{\mathsf{j}}}\) renames index \(\ensuremath{\mathsf{i}}\) to \(\ensuremath{\mathsf{j}}\).

Finally, this notation alleviates the duality problem, as can be seen in the definition of a RNN: \[\begin{aligned} x^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{emb}}[d]}} \\ h^{(t)} &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ A &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]} \times \ensuremath{\ensuremath{\mathsf{state'}}[d]}} \\ B &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]} \times \ensuremath{\ensuremath{\mathsf{emb}}[d]}} \\ c &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{state}}[d]}} \\ h^{(t+1)}_{\ensuremath{\mathsf{state}}} &= \tanh\left( A_{\ensuremath{\mathsf{state}},\ensuremath{\mathsf{state'}}} \, h^{(t)}_{\ensuremath{\mathsf{state}}} + B_{\ensuremath{\mathsf{state}},\ensuremath{\mathsf{emb}}} \, x^{(t)}_{\ensuremath{\mathsf{emb}}} + c_{\ensuremath{\mathsf{state}}} \right)\end{aligned}\] or equivalently, \[h^{(t+1)} = \tanh\left( A_{\ensuremath{\mathsf{state'}}} \cdot h^{(t)}_{\ensuremath{\mathsf{state}}} + B_{\ensuremath{\mathsf{emb}}} \cdot x^{(t)}_{\ensuremath{\mathsf{emb}}} + c \right)\]

Attention: \[\begin{aligned} \text{Att} &\colon \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq'}}[n']} \times\ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \times \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{key}}[d_k]}} \times \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq}}[n]} \times\ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \rightarrow \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{seq'}}[n']} \times\ensuremath{\ensuremath{\mathsf{val}}[d_v]}} \\ \text{Att}(Q,K,V) &= \softmax \left[ \frac{Q_{\ensuremath{\mathsf{key}}} \cdot K_\ensuremath{\mathsf{key}}}{\sqrt{d_k}} \right]_{\ensuremath{\mathsf{seq}}} \cdot V_{\ensuremath{\mathsf{seq}}}\end{aligned}\]

Multivariate normal distribution: \[\begin{aligned} X &\in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{batch}}[{b}]} \times \ensuremath{\ensuremath{\mathsf{d}}[{k}]}} \\ \mu &\in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{d}}[{k}]}}} \\ \Sigma & \in \mathbb{R}^{{\ensuremath{\ensuremath{\mathsf{d}}[k]} \times \ensuremath{\ensuremath{\mathsf{d'}}[k]}}} \\ {\cal N}(X; \mu, \Sigma) &= \frac{\displaystyle \exp\left(-\tfrac{1}{2} [X - \mu]_{\ensuremath{\mathsf{d}}}^\top \, \Sigma_{\ensuremath{\mathsf{d}},\ensuremath{\mathsf{d'}}}^{-1} \, [X - \mu]_{\ensuremath{\mathsf{d}}} \right)}{\sqrt{(2 \pi)^k \, \mathop{\text{det}} \Sigma_{\ensuremath{\mathsf{d}},\ensuremath{\mathsf{d'}}}}}\end{aligned}\]

Because this notation can be a little more verbose (often requiring you to write index names twice), we’d keep around the notation \(A \mathbin{\mathop{\boldsymbol\cdot}\limits_{\ensuremath{\mathsf{i}}}} B\) as a shorthand for \(A_{\ensuremath{\mathsf{i}}} \cdot B_{\ensuremath{\mathsf{i}}}\). We’d also keep named reductions, or at least \(\mathop{\mathrm{softmax}}\limits_{\ensuremath{\mathsf{i}}}\).

Indexing with a tensor of indices

Contributors: Tongfei Chen and Chu-Cheng Lin

NumPy defines two kinds of advanced (also known as fancy) indexing: by integer arrays and by Boolean arrays. Here, we generalize indexing by integer arrays to named tensors. That is, if \(A\) is a named tensor with \(D\) indices and \(I^1, \ldots, I^D\) are named tensors, called “indexers,” what is \(A_{I^1, \ldots, I^D}\)?

We first consider the case where all the indexers have the same shape \(\mathcal{S}\): \[\begin{aligned} A &\in F^{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}[X_1]}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$D$}}}[X_D]}} \\ I^d &\in X_D^{\mathcal{S}} & d &= 1, \ldots, D.\end{aligned}\] Then \(A_{I^1, \ldots, I^D}\) is the named tensor with shape \(\mathcal{S}\) such that for any \(s \in \tupleshape{\mathcal{S}}\), \[\begin{aligned} [A_{I^1, \ldots, I^D}]_s &= A_{I^1_s, \ldots, I^D_s}.\end{aligned}\] More generally, suppose the indexers have different but compatible shapes: \[\begin{aligned} A &\in F^{\ensuremath{\ensuremath{\mathsf{i_\text{$1$}}}[X_1]}, \ldots, \ensuremath{\ensuremath{\mathsf{i_\text{$D$}}}[X_D]}} \\ I^d &\in X_D^{\mathcal{S}_d} & d &= 1, \ldots, D,\end{aligned}\] where the \(\mathcal{S}_d\) are pairwise compatible. Then \(A_{I^1, \ldots, I^D}\) is the named tensor with shape \(\mathcal{S} = \bigcup_d \mathcal{S}_d\) such that for any \(s \in \tupleshape{\mathcal{S}}\), \[\begin{aligned} [A_{I^1, \ldots, I^D}]_s &= A_{I^1_{\left.s\right|_{\mathcal{S}_1}}, \ldots, I^D_{\left.s\right|_{\mathcal{S}_D}}}.\end{aligned}\]

Let’s consider a concrete example in natural language processing. Consider a batch of sentences encoded as a sequence of word vectors, that is, a tensor \(X \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{batch}}[B]}, \ensuremath{\ensuremath{\mathsf{sent}}[N]}, \ensuremath{\ensuremath{\mathsf{emb}}[E]}}\). For each sentence, we would like to take out the encodings of a particular span for each sentence \(b \in [B]\) in the batch, resulting in a tensor \(Y \in \mathbb{R}^{\ensuremath{\ensuremath{\mathsf{batch}}[B]}, \ensuremath{\ensuremath{\mathsf{span}}[M]}, \ensuremath{\ensuremath{\mathsf{emb}}[E]}}\).

We create a indexer for the \(\ensuremath{\mathsf{sent}}\) axis: \(I_{\ensuremath{\mathsf{sent}}} \in [N]^{\ensuremath{\mathsf{batch}}:B \times \ensuremath{\mathsf{span}}: M}\) that selects the desired tokens. Then we can write \[Y = X_{\ensuremath{\ensuremath{\mathsf{batch}}[I]}, \ensuremath{\ensuremath{\mathsf{sent}}[I]}, \ensuremath{\ensuremath{\mathsf{emb}}[I]}}\] where the other two indexers \[\begin{aligned} I_{\ensuremath{\mathsf{batch}}} &\in [B]^{\ensuremath{\ensuremath{\mathsf{batch}}[B]}} \\ [I_{\ensuremath{\mathsf{batch}}}]_{\ensuremath{\ensuremath{\mathsf{batch}}[b]}} &= b \\ I_{\ensuremath{\mathsf{emb}}} &\in [E]^{\ensuremath{\ensuremath{\mathsf{emb}}[E]}} \\ [I_{\ensuremath{\mathsf{sent}}}]_{\ensuremath{\ensuremath{\mathsf{sent}}[n]}} &= n\end{aligned}\] select all values of their respective indices.
